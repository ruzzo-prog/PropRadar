# Tester Three Level Pass Fail

## Когда применять

Применяй этот skill, если задача включает:

- QA-проверку после реализации;
- запрос на структурированный отчет `PASS/FAIL`;
- необходимость пройти тесты по уровням: `Unit -> Integration -> E2E`.

Не применяй для написания production-кода или архитектурного аудита.

## Трехуровневая стратегия

Всегда иди по порядку. Переход к следующему уровню только после стабильного результата текущего.

1. **Unit**
   - Проверяй локальную логику: функции, валидаторы, мапперы, edge-case поведение.
   - Цель: быстро отловить регрессии на уровне отдельных модулей.
2. **Integration**
   - Проверяй связки компонентов: API-слой, БД, очереди, вебхуки, контракты между модулями.
   - Цель: подтвердить, что взаимодействия работают корректно в реалистичном потоке.
3. **E2E**
   - Проверяй пользовательский или системный сценарий от входа до результата.
   - Цель: подтвердить, что целевой бизнес-сценарий выполняется целиком.

## Правила выполнения

1. Сначала определи охват: какие уровни обязательны для конкретного изменения.
2. Если уровень неприменим, пометь его как `SKIP` с явной причиной.
3. При первом `FAIL` фиксируй дефект и не маскируй итоговый статус.
4. После исправлений запускай повтор только затронутого уровня и зависимых уровней выше.
5. В отчете отделяй факты (команды/логи/симптом) от гипотез.

## Критерии PASS/FAIL

- `PASS`: все обязательные проверки уровня завершены успешно.
- `FAIL`: есть хотя бы одна обязательная неуспешная проверка.
- `SKIP`: уровень не запускался по валидной причине (например, нет UI для E2E).

Итоговый вердикт:

- `PASS`, если все обязательные уровни имеют `PASS`;
- `FAIL`, если хотя бы один обязательный уровень имеет `FAIL`.

## Формат отчета

Всегда возвращай отчет в единой структуре:

```markdown
QA Report: <краткое название проверки>
Overall Verdict: PASS | FAIL
Confidence: High | Medium | Low

Scope:
- Change: <что проверялось>
- Mandatory Levels: Unit, Integration, E2E

Level Results:
- Unit: PASS | FAIL | SKIP
  - Evidence: <ключевой лог/команда/факт>
  - Notes: <кратко>
- Integration: PASS | FAIL | SKIP
  - Evidence: <ключевой лог/команда/факт>
  - Notes: <кратко>
- E2E: PASS | FAIL | SKIP
  - Evidence: <ключевой лог/команда/факт>
  - Notes: <кратко>

Defects:
1. <дефект или "None">

Retest:
- Needed: Yes/No
- Plan: <что перезапустить>

Next Step:
- <например: "@documentor" при PASS или "@review" при FAIL>
```

## Минимальный чеклист перед ответом

- [ ] Все обязательные уровни отражены (PASS/FAIL/SKIP).
- [ ] Для каждого уровня есть Evidence.
- [ ] Overall Verdict согласован со статусами уровней.
- [ ] При FAIL перечислены дефекты и шаг ретеста.
- [ ] Указан следующий агентный шаг.

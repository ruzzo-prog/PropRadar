# Извлечение телефона (myhome.ge)

Цель этого контура — заполнить поле телефона лида источника **myhome** с минимальной нагрузкой на браузер.

## Порядок выполнения

1. **HTTP-first:** клиентские запросы к публичной странице карточки  
   `https://www.myhome.ge/{locale}/pr/{statement_id}/` (по умолчанию локаль **`ru`**).  
   Из HTML извлекается встроенный JSON:

   - тег **`__NEXT_DATA__`** (`application/json`);
   - по необходимости — блоки **`application/ld+json`** (schema.org).

   Найденное значение нормализуется до строки из **не менее 9 цифр** (пробелы и знаки `+` в исходнике отбрасываются при подсчёте длины). Сами номера в логи не пишутся.

2. **Playwright fallback:** если HTTP-слой вернул `None`, используется прежний сценарий **`MyHomePhoneEnricher`**: переход на карточку, подавление попапов, паузы как раньше, клик «показать номер», разбор ответа **`phone/show`**.

Очередность и интервалы браузера (в частности паузы на странице и ожидания сети) **не должны произвольно сокращаться** при изменениях этого контура без отдельного решения (rate behaviour / стабильность под reCAPTCHA).

## Связанные файлы кода

- `src/parsers/adapters/myhome/phone_extractor.py` — `get_phone()`, чистые парсеры HTML.
- `src/parsers/adapters/myhome/phone.py` — оркестрация HTTP prefetch + fallback.
- `tests/unit/test_phone_extractor.py` — юнит-покрытие парсеров и нормализации (без live myhome).

## Проверки (зафиксировано в релизном контуре)

- **`pytest tests/unit/test_phone_extractor.py`** — **20 passed** (сессия 2026-05-09).
- Полный **Scanner** и интеграция с live API на этом проходе — **SKIP**; регресс браузерного fallback при необходимости — отдельный прогон воркера / Scanner.

## Оркестрация деплоя

Вызов **`playwright-worker`** с телом **`{"adapter":"myhome","phase":"phone"}`** и ответом **HTTP 202** остаётся без изменений: воркер по-прежнему запускает тот же обогатитель; различие только в том, что часть лидов завершается на HTTP до открытия браузера.
